Surface Explain it like I am 10 What happens in the game Why it matters in ATV StarterKit
Secrets These are the keys to the fort. Secret leaks push infection into nearby tools and the app. `atv-security` looks for hardcoded tokens and other exposed secrets.
MCP This is the robot's tool belt. Wildcard MCP access lets trouble jump further than it should. The real scan checks risky MCP scope and unsafe config patterns.
Hooks These are chores that run automatically. Hook exfiltration can quietly leak data while work happens. The real scan checks hook scripts and setup steps for unsafe execution.
Skills and Agents These are the helper's instructions and specialist workers. Prompt injection or bad access chains twist the helper into doing the wrong thing. `atv-security` reviews instruction safety, hidden injections, and risky agent access.
VS Code / Permissions These are the room rules. Unsafe trust or auto-approve drift makes outbreaks spread faster. The real scan checks permission posture and suspicious extension guidance.
App + OWASP / STRIDE This is the actual product the team ships. Later waves pressure the app with SSRF, injection, and privilege chains. The unified skill also checks source code with OWASP Top 10 and STRIDE thinking.
The big idea: one bad setting can help another bad setting spread. That is why `atv-security` matters. It gives one pass across config, instructions, hooks, permissions, and app risk so you can stop chains before they become an outage.
